# exp_006：多任务 CVR 建模（Ali-CCP，分层采样+统计特征+DirectCTCVR对比）

## 实验信息

| 项目 | 内容 |
|------|------|
| 日期 | 2026-03-25 |
| 数据集 | Ali-CCP（天池 408） |
| 采样策略 | click=0 保留5%，click=1,buy=0 保留50%，buy=1 全保留 |
| 样本量 | train=257万，val=29万，test=290万 |
| 稀疏特征 | 15个（f_101~f_301） |
| 数值特征 | **13个**（用户/广告/类目统计） |
| Epochs | 1 |
| Batch size | 4096 |

## 统计特征（13个）

| 特征 | 说明 |
|------|------|
| u_imp_cnt | 用户总曝光次数（log1p + z-score） |
| u_click_cnt | 用户总 click 次数（log1p + z-score） |
| u_buy_cnt | 用户总 buy 次数（log1p + z-score） |
| u_ctr | 用户 click/imp 比率 |
| u_cvr | 用户 buy/click 比率 |
| u_ctcvr | 用户 buy/imp 比率 |
| ad_imp_cnt | 广告总曝光次数（log1p + z-score） |
| ad_click_cnt | 广告总 click 次数（log1p + z-score） |
| ad_ctr | 广告 CTR |
| ad_cvr | 广告 CVR（buy/click） |
| ad_ctcvr | 广告 CTCVR（buy/imp） |
| cate_ctr | 类目 CTR |
| cate_ctcvr | 类目 CTCVR |

## 实验结果

| 模型 | CTR AUC | CVR AUC | CTCVR AUC | 训练时间 |
|------|---------|---------|-----------|---------|
| DirectCTCVR（单任务基线） | — | — | 0.4872 | 395s |
| Shared-Bottom | 0.5459 | 0.5090 | 0.4817 | 400s |
| ESMM | 0.5460 | 0.5526 | 0.5083 | 399s |
| MMoE | 0.5459 | 0.5113 | 0.4843 | 409s |
| **ESCM2** | 0.5459 | 0.5438 | **0.5127** | 399s |

## 与 exp_004（无统计特征）对比

| 指标 | exp_004（无统计特征） | exp_006（+统计特征） | 变化 |
|------|---------------------|---------------------|------|
| CTR AUC（最佳） | 0.5481 | 0.5460 | -0.2千分点（持平） |
| CVR AUC（最佳） | 0.5790 | 0.5526 | -2.6千分点 |
| CTCVR AUC（最佳） | 0.5365 | 0.5127 | -2.4千分点 |

## 关键发现

1. **Ali-CCP 统计特征效果不如预期**：加了 13 个统计特征后 AUC 反而略有下降
   - 可能原因：Ali-CCP 的统计特征存在严重的数据泄露问题——全量计算统计量时用了测试集的数据
   - 需要用时序切分：用早期数据的统计量作为特征，预测晚期数据的标签
2. **DirectCTCVR 单任务 CTCVR=0.4872**，低于多任务模型，说明多任务学习对 Ali-CCP 确实有帮助
3. **ESCM2 CTCVR 最优（0.5127）**，反事实正则化在稀疏 buy 信号下有效
4. **对比 Taobao（CTCVR=0.95）**，Ali-CCP 整体 AUC 仍然很低，数据集本身限制较大

## 注意：数据泄露风险

全量统计特征（u_ctr、ad_ctcvr 等）是基于全量 skeleton 计算的，包含了测试集的行为信息。
正确做法：按时间切分，用前 T 天的统计量作为特征，预测第 T+1 天的标签。
Ali-CCP 数据集包含时间戳，可以实现时序切分。
