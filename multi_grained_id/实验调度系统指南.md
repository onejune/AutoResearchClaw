## 系统概述

### 什么是实验调度系统？

实验调度系统是一个自动化的实验管理平台，专为多 GPU/CPU 环境设计，能够：

- ✅ **自动队列管理** - 按优先级自动启动待运行实验
- 🔄 **实时监控** - 跟踪实验进度、检测异常
- 🛡️ **失败恢复** - 自动重启意外失败的实验（最多 3 次）
- 🎯 **智能调度** - 根据 GPU/CPU 负载自动分配资源
- 📊 **完整日志** - 记录所有实验的运行状态和结果

## 核心功能

### 1. 自动调度

守护进程会定期检查实验队列，自动启动待运行的实验：

```python
# 调度逻辑（简化版）
while True:
    # 1. 检查是否有待运行的实验
    pending = get_pending_experiments()
    
    # 2. 检查可用资源
    available_gpus = check_gpu_availability()
    available_cpu = check_cpu_availability()
    
    # 3. 按优先级分配资源
    for exp in sorted(pending, key=lambda e: e.priority):
        if has_resource(available_gpus, available_cpu):
            start_experiment(exp)
    
    sleep(interval_seconds)
```

### 2. 失败恢复

检测到实验意外停止时，自动重启：

```
实验崩溃 → 检测到 PID 消失 → 等待 5 秒 → 重启实验 → 记录重启次数
                                    ↓
                            重启次数 >= 3?
                               ↓        ↓
                             YES       NO
                               ↓        ↓
                         标记为失败   继续监控
```

### 3. 进度追踪

实时解析实验日志，显示训练进度：

```
Epoch 1:   5%|▌         | 2882/6378 [02:15<45:30, 12.3it/s] loss=0.4408
```

## 守护进程管理

### 启动方式对比

| 方式 | 命令 | 适用场景 |
|------|------|----------|
| **管理脚本** | `./scripts/manage_experiments.sh start` | 日常使用，推荐 |
| **nohup** | `nohup python experiment_manager.py --daemon &` | 需要自定义参数 |
| **screen/tmux** | `screen -S exp && python experiment_manager.py --daemon` | 需要交互式调试 |

### 管理脚本命令详解

```bash
# start   - 启动守护进程
#          检查是否已在运行，避免重复启动
./scripts/manage_experiments.sh start

# stop    - 停止守护进程
#          优雅终止进程，清理 PID 文件
./scripts/manage_experiments.sh stop

# restart - 重启守护进程
#          先 stop 再 start，中间等待 2 秒
./scripts/manage_experiments.sh restart

# status  - 查看运行状态
#          显示 PID 和最近 20 行日志
./scripts/manage_experiments.sh status

# log     - 查看日志
#          显示最近 50 行日志
./scripts/manage_experiments.sh log
```

### 自定义守护进程参数

```bash
# 修改检查间隔（默认 30 秒）
nohup python experiment_manager.py --daemon --interval 60 > /tmp/exp_manager.log 2>&1 &

# 更短的间隔适合快速实验，更长的间隔节省资源
```

## 实验配置

### 配置文件结构

`experiments_config.json` 定义了所有待管理的实验：

```json
{
  "exp001": {
    "name": "AutoEmb",
    "script": "scripts/run_exp001_autoemb.py",
    "priority": 1,
    "description": "自动化维度分配"
  },
  "exp002": {
    "name": "DDS",
    "script": "scripts/run_exp002_dds.py",
    "priority": 2,
    "description": "数据分布搜索"
    }
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 实验显示名称 |
| `script` | string | ✅ | 实验脚本路径（相对项目根目录） |
| `priority` | int | ✅ | 优先级（数字越小越优先） |
| `description` | string | ❌ | 描述信息 |

### 添加新实验

1. **创建实验脚本**
   ```bash
   cat > scripts/run_exp015_newmethod.py << 'EOF'
   #!/usr/bin/env python3
   """实验 015: 新方法"""
   # ... 实验代码 ...
   EOF
   chmod +x scripts/run_exp015_newmethod.py
   ```

2. **编辑配置文件**
   ```bash
   # 方法一：手动编辑
   vim experiments_config.json
   
   # 方法二：使用 Python
   python << 'EOF'
   import json
   with open('experiments_config.json', 'r') as f:
       config = json.load(f)
   config['exp015'] = {
       'name': 'NewMethod',
       'script': 'scripts/run_exp015_newmethod.py',
       'priority': 15,
       'description': '新方法测试'
   }
   with open('experiments_config.json', 'w') as f:
       json.dump(config, f, indent=2, ensure_ascii=False)
   EOF
   ```

3. **重启守护进程**
   ```bash
   ./scripts/manage_experiments.sh restart
   ```

### 修改优先级

优先级数字越小，越先运行：

```json
{
  "exp001": {"priority": 1},   // 最高优先级
  "exp002": {"priority": 5},   // 中等优先级
  "exp015": {"priority": 100}  // 低优先级
}
```

## 资源调度策略

### GPU 调度

**空闲判断标准**：
- GPU 利用率 < 10%
- 显存使用 < 500MB

**分配策略**：
1. 优先选择完全空闲的 GPU
2. 多个空闲 GPU 时选择编号最小的
3. 支持多 GPU 并行运行多个实验

### CPU 回退

当所有 GPU 都忙碌时：

```
GPU 全忙 → 检查 CPU 利用率 → < 70%? → 是 → 使用 CPU 运行
                                   ↓
                                  否
                                   ↓
                              等待 GPU 释放
```

### 并发控制

系统会自动控制并发数：

- **GPU 实验**：每个 GPU 同时只运行 1 个实验
- **CPU 实验**：最多同时运行 2 个（避免 CPU 过载）

### 查看资源状态

```bash
# 使用系统命令
nvidia-smi               # GPU 状态
top -b -n 1              # CPU/内存状态

# 使用实验管理器
python experiment_manager.py --status
```

## 故障排查

### 常见问题

#### 1. 守护进程未启动

**症状**：`./scripts/manage_experiments.sh status` 显示未运行

**解决**：
```bash
# 检查 PID 文件是否残留
rm -f /tmp/experiment_manager.pid

# 手动启动
./scripts/manage_experiments.sh start

# 查看日志
tail -20 /tmp/experiment_manager.log
```

#### 2. 实验无法启动

**症状**：实验一直停留在 `pending` 状态

**可能原因**：
- 没有空闲 GPU/CPU
- 实验脚本不存在或无执行权限
- 依赖库缺失

**解决**：
```bash
# 检查脚本
ls -la scripts/run_expXXX.py
chmod +x scripts/run_expXXX.py

# 检查依赖
python -c "import torch; import transformers; print('OK')"

# 手动运行实验（看错误信息）
python scripts/run_expXXX.py
```

#### 3. 实验频繁重启

**症状**：`restart_count` 不断增加

**可能原因**：
- 代码 bug 导致崩溃
- 显存不足
- 数据加载失败

**解决**：
```bash
# 查看详细日志
cat /tmp/exp_logs/expXXX.log | tail -100

# 检查显存
nvidia-smi

# 手动运行复现问题
python scripts/run_expXXX.py
```

#### 4. GPU 未被识别

**症状**：所有 GPU 显示为忙碌或不可用

**解决**：
```bash
# 检查 nvidia-smi
nvidia-smi

# 检查 CUDA_VISIBLE_DEVICES
echo $CUDA_VISIBLE_DEVICES

# 重启管理器
./scripts/manage_experiments.sh restart
```

### 重置系统状态

如果系统状态混乱，可以完全重置：

```bash
# 1. 停止守护进程
./scripts/manage_experiments.sh stop

# 2. 删除状态文件
rm -f experiment_manager_state.json

# 3. 清理日志（可选）
rm -rf /tmp/exp_logs/*

# 4. 重新启动
./scripts/manage_experiments.sh start
```

## 最佳实践

### 1. 始终使用守护进程

不要手动逐个启动实验，让调度系统自动管理：

```bash
# ✅ 推荐
./scripts/manage_experiments.sh start

# ❌ 不推荐
python scripts/run_exp001.py &
python scripts/run_exp002.py &
python scripts/run_exp003.py &
```

### 2. 合理设置优先级

- **重要实验**：priority 1-10
- **常规实验**：priority 11-50
- **低优先级实验**：priority 51-100

### 3. 定期检查日志

每天至少检查一次日志，及时发现异常：

```bash
# 快速检查
./scripts/manage_experiments.sh log | grep -i "error\|fail"
```

### 4. 监控资源使用

避免资源争用：

```bash
# 查看 GPU 使用情况
watch -n 5 nvidia-smi

# 查看 CPU/内存
htop
```

### 5. 备份状态文件

定期备份状态文件，防止数据丢失：

```bash
# 每日备份
cp experiment_manager_state.json experiment_manager_state.$(date +%Y%m%d).json
```

### 6. 实验命名规范

使用清晰的实验 ID 和名称：

```json
{
  "exp015": {
    "name": "Hierarchical_v3_Attention",  // 清晰描述
    "description": "层次化 Embedding v3，使用 Attention 机制"
  }
}
```

### 7. 集成到监控告警

可以将状态检查集成到 crontab：

```bash
# 每天早上 9 点发送实验状态报告
0 9 * * * cd /mnt/workspace/open_research/autoresearch/multi_grained_id && \
          python experiment_manager.py --status >> /var/log/exp_report.log 2>&1
```
