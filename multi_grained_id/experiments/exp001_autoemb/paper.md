# exp001: AutoEmb - 自动化 Embedding 维度分配

## 对应 Paper

**AutoEmb: Automated Embedding Dimensionality Search for Deep Learning Recommendation Models**

- **会议**: KDD 2021
- **作者**: Jieming Zhu et al. (Huawei)
- **链接**: https://dl.acm.org/doi/10.1145/3447548.3467265

## 核心思想

### 问题动机

传统推荐系统中，所有 ID 特征（用户 ID、物品 ID、广告 ID 等）的 embedding 维度是**人工统一设定**的（如全部 64 维）。但这存在两个问题：

1. **样本不平衡**：头部 ID 样本百万级，尾部 ID 只有几个样本
   - 统一高维 → 尾部 ID 过拟合
   - 统一低维 → 头部 ID 欠拟合

2. **参数效率低**：embedding 表通常占模型总参数的 70%~90%，但很多维度是冗余的

### 方法概述

AutoEmb 提出**自动化为每个特征搜索最优 embedding 维度**：

```
┌─────────────────────────────────────┐
│  Feature    │ Vocab Size │ Dim      │
├─────────────────────────────────────┤
│  adid       │ 10,000     │ 128      │ ← 高频，高维
│  bundle     │ 50,000     │ 64       │ ← 中频，中维
│  city       │ 500        │ 16       │ ← 低频/密集，低维
└─────────────────────────────────────┘
```

### 关键技术

#### 1. 维度搜索空间

为每个特征定义可选维度集合：`D = {8, 16, 32, 64, 128}`

#### 2. 强化学习搜索器

- **State**: 当前已选维度组合 + 验证集性能
- **Action**: 为下一个特征选择维度
- **Reward**: 验证集 AUC - λ × 参数量（平衡性能与效率）

#### 3. 渐进式训练

- 先在子集上快速搜索维度配置
- 再用最优配置在全量数据上训练最终模型

### 简化实现（本项目）

由于完整 RL 搜索成本较高，本项目采用**启发式规则**近似：

| 条件 | 推荐维度 |
|------|----------|
| vocab > 1M 或 中位频次 < 10 | 16 维（超稀疏） |
| vocab > 100k 或 中位频次 < 100 | 32 维（稀疏） |
| vocab > 10k | 64 维（中等） |
| vocab ≤ 1k | 32 维（密集） |

## 预期收益

- **参数量减少**：30%~50%（相比固定 64 维）
- **AUC 持平或微升**：尾部 ID 泛化能力提升
- **训练速度提升**：embedding 查找和梯度更新更快

## 实验设计

### Baseline
- 所有 ID 特征统一 64 维

### AutoEmb (启发式)
- 按上述规则自动分配维度

### 评估指标
- AUC / PCOC（整体 + 分 BT）
- 参数量（MB）
- 训练时间（秒/epoch）
