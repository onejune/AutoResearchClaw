## 结论

### H1: 去掉虚假相关特征后 OOD AUC 提升 ❌ 不成立
- 方案 B AUC = 0.7487，比 A (0.7539) 下降了 0.52 个千分点
- bt_16 AUC 反而下降 (0.5110 → 0.4886)
- 去掉这 3 个特征对整体性能有轻微负面影响

### H2: 只保留因果特征后 PCOC 方差减小 ❌ 不成立
- 方案 C PCOC std = 0.2710，与 A (0.2706) 几乎相同
- 但整体 AUC 大幅下降 7.5 个千分点
- 特征过少导致表达能力不足

### H3: 特征子集选择比 IRM 更有效 ❌ 不成立
- 方案 B/C/D 的 AUC 均低于 IRM λ=0 (0.7539)
- 特征子集选择方法在本场景下效果有限

### 关键洞察
1. **虚假相关特征可能不是主要问题**：去掉后 AUC 反而下降，说明这些特征在 ID 上仍有预测价值
2. **因果特征集太小**：17 个特征不足以支撑模型表达能力，AUC 大幅下降
3. **bt_16 仍是最大难点**：无论哪种方案，bt_16 AUC 都在 0.46~0.52 之间，接近随机
4. **PCOC 改善有限**：所有方案的 PCOC 方差都在 0.27 左右，没有显著收窄

### 下一步建议
- **bt_16 专项分析**：该 BT 正样本率 66.1%，与训练集 (42%) 差异过大，需要 domain adaptation
- **更精细的特征选择**：尝试逐步增加特征数量 (30→50→70→90)，找到最佳平衡点
- **其他 OOD 方法**：Domain adversarial training、CORAL、MMD 等
