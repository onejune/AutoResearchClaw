## 结论

### H1: IRM 能提升 OOD 整体 AUC ❌ 不成立
- 随 λ 增大，AUC 单调下降：0.7539 → 0.7465 → 0.7459 → 0.7317
- IRM 惩罚损害了整体预测能力

### H2: IRM 能减小跨 BT 的 PCOC 方差 ❌ 不成立
- PCOC std 随 λ 增大反而略有上升（0.2706 → 0.2984）
- 校准稳定性没有改善

### H3: IRM 对 bt_16 (完全失效 BT) 有特殊改善 ⚠️ 轻微改善但不显著
- bt_16 AUC：0.5110 (λ=0) → 0.5024 (λ=1) → 0.5194 (λ=10) → 0.5237 (λ=100)
- λ=100 时 bt_16 AUC 略有提升，但整体 AUC 代价过大

### 总体评价
- **标准 IRMv1 在本场景下效果不佳**，可能原因：
  1. 环境划分粒度不够（3个环境，每个环境内部 BT 仍有异质性）
  2. IRM 惩罚项数值极小（~0.0003），对梯度影响有限
  3. 1 epoch 训练，IRM 惩罚还没充分发挥作用
  4. bt_16 分布漂移过于极端（正样本率 66.1% vs 训练集 11.4%），非 IRM 能解决的范畴

### 下一步建议
- **Phase 3**: 特征子集选择——去掉虚假相关特征（`os`, `duf_inner_dev_pkg_pur_*`）后 AUC/PCOC 变化
- **改进 IRM**: 更细粒度环境划分（每个 BT 一个环境），或换用 IRM-Games / V-REx 等变体
- **bt_16 专项**: 该 BT 正样本率 66.1%，与训练集差异过大，考虑 domain adaptation 或单独建模
